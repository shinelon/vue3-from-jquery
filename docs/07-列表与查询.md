## 一、先看 jQuery 怎么做表格（复习痛点）

用 jQuery 做一个带分页的用户列表，你大概要写这么多：

```js
$.ajax({
  url: '/api/users',
  success: function (data) {
    // 1. 手动拼 <tr><td>...</td></tr>
    var html = ''
    data.forEach(function (u) {
      html += '<tr><td>' + u.name + '</td><td>' + u.email + '</td></tr>'
    })
    $('#tbody').html(html)            // 2. 手动塞进 DOM
    renderPagination(data.length)      // 3. 手动算分页、手动画页码
  }
})

$('#search').on('input', function () {
  var kw = $(this).val()
  // 4. 手动过滤、手动重新渲染整个表格
})
```

痛点：**手动拼 HTML、手动塞 DOM、手动算分页、手动重渲染**。搜索一改，整个表格你都得重新拼一遍。

Vue + Element Plus 怎样？**一个 `<el-table>` 标签 + 一份响应式数据**，表格自动渲染；数据一变，表格自动更新；分页、搜索都是改个变量的事。

## 二、`el-table`：声明式表格

核心就两步：**给数据，声明列**。

```vue
<el-table :data="users" border stripe>
  <el-table-column prop="name" label="姓名" />
  <el-table-column prop="email" label="邮箱" />
</el-table>
```

- `:data="users"`：表格的数据源（一个数组）；
- `<el-table-column prop="name" label="姓名" />`：一列，`prop` 取对象的哪个字段，`label` 是表头。

**就这么多**。表格、表头、斑马纹（`stripe`）、边框（`border`）全自动。`users` 一变，表格立刻刷新——你**完全不用碰 DOM**。

| jQuery | Vue + Element Plus |
|---|---|
| `forEach` 拼 `<tr><td>` + `.html()` 塞进去 | `<el-table :data>` + `<el-table-column>` |
| 数据变了手动重渲染 | 数据变了自动更新 |

## 三、作用域插槽：自定义列内容

有些列不是简单显示字段，要"加工"（比如角色显示成中文标签、状态用颜色）。用**作用域插槽**：

```vue
<el-table-column label="角色">
  <template #default="{ row }">
    <el-tag>{{ roleMap[row.role] }}</el-tag>
  </template>
</el-table-column>
```

`#default="{ row }"` 是关键：`row` 就是**当前这一行的数据对象**（`#default` 是插槽，`{ row }` 是 ES6 解构，第 5 章讲过插槽、ES6 速查讲过解构）。拿到 `row`，你想怎么渲染都行——标签、按钮、图片、甚至另一个组件。

操作列也是这么写的：

```vue
<el-table-column label="操作">
  <template #default="{ row }">
    <el-button size="small" @click="handleDelete(row)">删除</el-button>
  </template>
</el-table-column>
```

## 四、把数据请求接上（axios + onMounted + loading）

数据从哪来？请求后端（第 6 章搭好的 json-server）：

```js
import { ref, onMounted } from 'vue'
import { getUsers } from '../api/user'
import { ElMessage } from 'element-plus'

const users = ref([])
const loading = ref(false)

async function loadData() {
  loading.value = true
  try {
    users.value = await getUsers()    // 拦截器已取出 data，直接是数组
  } catch (e) {
    ElMessage.error('加载失败：' + e.message)   // 错误提示
  } finally {
    loading.value = false
  }
}

onMounted(loadData)    // 组件挂载后自动加载（类似 jQuery 的 $(function(){})）
```

- **`onMounted`**：组件挂载到页面后执行的钩子，常用来发首屏请求。对照 jQuery 的 `$(document).ready()`。
- **`loading`**：用 `v-loading="loading"` 绑到表格上，请求中自动转圈，结束消失——不用自己写遮罩。
- **`ElMessage`**：Element Plus 的全局提示，对照 jQuery 时代的 `alert()`，但好看得多、不阻塞。

## 五、分页（前端分页）

入门用**前端分页**：一次请求全部数据，前端用 `computed` 切片。简单直观：

```js
import { computed } from 'vue'

const currentPage = ref(1)      // 当前页
const pageSize = ref(5)         // 每页几条

// 当前页要显示的数据（切片）
const pagedUsers = computed(() => {
  const start = (currentPage.value - 1) * pageSize.value
  return filtered.value.slice(start, start + pageSize.value)
})
```

模板里放一个分页器：

```vue
<el-pagination
  v-model:current-page="currentPage"     <!-- 双向绑定当前页 -->
  v-model:page-size="pageSize"           <!-- 双向绑定每页条数 -->
  :total="filtered.length"               <!-- 总条数 -->
  :page-sizes="[5, 10, 20]"
  layout="total, sizes, prev, pager, next"
/>
```

`currentPage` 一变，`pagedUsers` 自动重算，表格自动刷新——**又是"改数据，视图自动变"**。

> 💡 **关于前端 vs 后端分页**：数据少时前端分页够用（一次全取，前端切片）。真实项目数据成千上万时，会改成**后端分页**：请求时传 `_page`、`_limit`，后端只返回那一页。原理一样，只是切片的活儿交给后端。入门先掌握前端分页，理解了机制，后端分页只是换个参数。

## 六、搜索过滤（computed）

搜索框 + `computed` 过滤，依然是"改数据自动变"：

```js
const keyword = ref('')

const filtered = computed(() => {
  const kw = keyword.value.trim().toLowerCase()
  if (!kw) return users.value
  return users.value.filter(u =>
    u.name.toLowerCase().includes(kw) ||
    u.email.toLowerCase().includes(kw)
  )
})
```

```vue
<el-input v-model="keyword" placeholder="搜索姓名/邮箱" clearable @input="currentPage = 1" />
```

`@input="currentPage = 1"`：输入时回到第 1 页（否则搜索后可能停在一个空页）。注意 `pagedUsers` 依赖的是 `filtered`，而 `filtered` 依赖 `keyword`——**`keyword` 变 → `filtered` 变 → `pagedUsers` 变 → 表格变**，这条链全是自动的。

## 七、删除（带确认）

删除用 `DELETE /users/:id`。先弹确认框，用户确认了再删：

```js
import { ElMessageBox } from 'element-plus'
import { deleteUser } from '../api/user'

async function handleDelete(row) {
  // 1. 弹确认框（返回 Promise，点取消会 reject）
  try {
    await ElMessageBox.confirm(`确定删除「${row.name}」吗？`, '提示', { type: 'warning' })
  } catch {
    return    // 用户点了取消，直接返回
  }
  // 2. 确认了，发删除请求
  await deleteUser(row.id)
  ElMessage.success('删除成功')
  loadData()   // 重新加载列表
}
```

`api/user.js` 里加：

```js
export function deleteUser(id) {
  return request.delete(`/users/${id}`)
}
```

| jQuery | Vue |
|---|---|
| `confirm()`（丑、阻塞）+ `$.ajax({ type:'DELETE' })` | `ElMessageBox.confirm`（好看、Promise）+ `request.delete` |
