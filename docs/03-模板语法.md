# 第 3 章 · 模板语法：v-if / v-for / v-model / v-on

前两章你已经零散用过 `{{ }}`、`v-model`、`@click`、`v-for`。这一章我们把模板里**所有常用指令**系统过一遍——每一条都对照 jQuery 你熟悉的做法。

> 📂 可运行 demo 在 [`demos/ch03-template/`](../demos/ch03-template/)，一个带筛选的任务列表，综合用到了本章大部分指令。
>
> 💡 这一章内容多，但都是"查表即用"的东西。**不用全背**，当字典用，写的时候回来翻就行。

---

## 一、文本插值 `{{ }}`

```vue
<p>{{ msg }}</p>
<p>{{ count + 1 }}</p>              <!-- 支持表达式 -->
<p>{{ ok ? '是' : '否' }}</p>      <!-- 三元 -->
```

| jQuery | Vue |
|---|---|
| `$('#p').text(msg)` | `{{ msg }}` |

`{{ }}` 里只能写**表达式**（有返回值的），不能写语句（`if`、`for`、`let`）。

---

## 二、属性绑定 `v-bind`（简写 `:`）

```vue
<img :src="url" />                 <!-- v-bind:src 简写成 :src -->
<a :href="link">链接</a>
<button :disabled="!canSubmit">提交</button>
<div :id="'box-' + index"></div>
```

| jQuery | Vue |
|---|---|
| `$('img').attr('src', url)` | `:src="url"` |
| `$('button').prop('disabled', true)` | `:disabled="true"` |

记住简写：`v-bind:xxx` → `:xxx`，实际项目里几乎只用简写。

---

## 三、事件 `v-on`（简写 `@`）

```vue
<button @click="add">加</button>            <!-- v-on:click 简写成 @click -->
<input @keyup="onKey" />
```

| jQuery | Vue |
|---|---|
| `$('button').on('click', fn)` | `@click="fn"` |

### 事件修饰符（省掉手写的 e.xxx）

```vue
<button @click.stop="fn">...</button>       <!-- 阻止冒泡，相当于 e.stopPropagation() -->
<form @submit.prevent="save">...</form>     <!-- 阻止默认，相当于 e.preventDefault() -->
<input @keyup.enter="submit" />             <!-- 只有按回车才触发 -->
<input @keyup.esc="cancel" />               <!-- 只有按 Esc 才触发 -->
```

对照 jQuery，你本来得在回调里写 `if (e.key === 'Enter')`、`e.stopPropagation()`——Vue 用修饰符一行搞定，更直观。

---

## 四、条件渲染 `v-if` / `v-else` / `v-show`

```vue
<p v-if="count > 10">多于 10</p>
<p v-else-if="count > 5">多于 5</p>
<p v-else>很少</p>

<div v-show="visible">我一直在 DOM 里，只是被 display:none 了</div>
```

| jQuery | Vue |
|---|---|
| `if` 判断后 `.show()/.hide()` | `v-if` / `v-show` |

### `v-if` vs `v-show` 的区别（常考）

| | `v-if` | `v-show` |
|---|---|---|
| false 时 | **真的从 DOM 删掉** | 元素还在，只是 `display:none` |
| 切换成本 | 高（反复创建/销毁） | 低（只切 CSS） |
| 适用场景 | 不常切换 | **频繁**切换（如 Tab） |

---

## 五、列表渲染 `v-for`

```vue
<li v-for="(item, index) in list" :key="item.id">
  {{ index }} - {{ item.name }}
</li>
```

| jQuery | Vue |
|---|---|
| `$.each(list, function(i, item){ ... })` + 拼 `<li>` | `v-for` |

### ⚠️ `key` 很重要

每个 `v-for` 项都要给一个唯一的 `:key`（通常用数据的 `id`）：

```vue
<li v-for="item in list" :key="item.id">...</li>   <!-- ✅ 用 id -->
<li v-for="item in list" :key="index">...</li>     <!-- ❌ 别用 index -->
```

**为什么不能用 `index` 当 key？** 列表增删时，index 会错位，Vue 可能复用错元素，导致渲染混乱（比如输入框内容串到别的项上）。用稳定的 `id` 才安全。

---

## 六、双向绑定 `v-model`

```vue
<input v-model="msg" />                        <!-- 文本框 -->
<textarea v-model="content"></textarea>
<select v-model="city"><option>...</option></select>
<input type="checkbox" v-model="agree" />      <!-- 单个：布尔 -->
<input type="checkbox" value="A" v-model="tags" /> <!-- 多个：数组 -->
```

| jQuery | Vue |
|---|---|
| `$('#i').val()` 读 + `.val(x)` 写 + 自己监听 `change` 同步 | `v-model` 一行双向同步 |

### 修饰符

```vue
<input v-model.lazy="msg" />      <!-- 失焦/回车才同步（默认是 input 即同步） -->
<input v-model.number="age" />    <!-- 自动转成数字 -->
<input v-model.trim="name" />     <!-- 自动去首尾空格 -->
```

> `v-model` 本质是 `:value` + `@input` 的语法糖。第 1 章的 Todo 输入框就是靠它自动同步的。

---

## 七、`v-html`（渲染富文本，慎用）

```vue
<div v-html="rawHtml"></div>
```

| jQuery | Vue |
|---|---|
| `$('#d').html(htmlStr)` | `v-html="rawHtml"` |

⚠️ **XSS 风险**：`v-html` 会把字符串当 HTML 解析。如果内容来自用户输入，可能被注入恶意脚本。**只在内容可信时用**，绝大多数情况用 `{{ }}` 就够。

---

## 八、`class` / `style` 绑定

```vue
<!-- 对象语法：键是类名，值是布尔（为真才加这个类） -->
<div :class="{ active: isActive, 'text-danger': hasError }"></div>

<!-- 数组语法 -->
<div :class="['btn', isActive && 'btn-active']"></div>

<!-- style 绑定（驼峰命名） -->
<div :style="{ color: textColor, fontSize: size + 'px' }"></div>
```

| jQuery | Vue |
|---|---|
| `.addClass('active') / .removeClass('active')` | `:class="{ active: isActive }"` |
| `.css('color', 'red')` | `:style="{ color: textColor }"` |

Vue 的好处：**类名跟数据绑定，数据变，样式自动变**——不用手动 `addClass/removeClass`。

## 九、综合 demo：带筛选的任务列表

打开 [`demos/ch03-template/index.html`](../demos/ch03-template/index.html)。它把这些指令都用上了：

```vue
<!-- 筛选：v-model 单选 -->
<label><input type="radio" value="all" v-model="filter" /> 全部</label>
<label><input type="radio" value="todo" v-model="filter" /> 未完成</label>
<label><input type="radio" value="done" v-model="filter" /> 已完成</label>

<!-- 列表：v-for + :key + :class 条件样式 -->
<li v-for="t in filtered" :key="t.id" :class="{ done: t.done }">
  <input type="checkbox" v-model="t.done" />   <!-- v-model 绑对象属性 -->
  <span>{{ t.text }}</span>
  <button @click="remove(t.id)">删除</button>   <!-- @click -->
</li>

<script>
// 用 computed 根据 filter 算出过滤后的列表（复习第 2 章）
const filtered = computed(() => {
  if (filter.value === 'todo') return todos.value.filter(t => !t.done)
  if (filter.value === 'done') return todos.value.filter(t => t.done)
  return todos.value
})
</script>
```

试着：勾选/取消任务（`v-model` 改对象属性，划线样式 `:class` 自动跟着变）、切换筛选（`filtered` 自动重算）——**全程没碰 DOM**。
