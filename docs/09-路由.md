# 第 9 章 · 路由 Vue Router：多页面

到目前为止我们的项目只有一个页面（用户列表）。真实后台通常有**多个页面**：列表页、详情页、编辑页……这一章我们用 **Vue Router** 给项目加上多页面能力。

> 📂 本章改动 [`project/user-admin/`](../project/user-admin/)：新增 `src/router/index.js`、`src/views/UserDetail.vue`，更新 `main.js`、`App.vue`、`UserList.vue`、`api/user.js`。对应阶段 `ch09-router`。

---

## 一、多页面的两种方式（SPA vs 传统）

先建立认知，这是 jQuery 程序员最需要转变的一点。

**传统方式（jQuery 时代）**：每个页面一个 `.html` 文件。跳转靠 `location.href = 'detail.html'`——**整页刷新**，白屏一下，重新加载所有资源。

**单页应用（SPA，Vue/React 的方式）**：**只有一个 html 文件**。所谓"多个页面"，其实是**不同的组件**，靠 JS 在同一个页面里切换。切换时**不刷新整页**，只换中间那块内容——快、流畅、像原生 App。

负责"根据网址显示哪个组件"的，就是**路由（Router）**。Vue 官方的路由库叫 **Vue Router**。

| 传统（jQuery） | SPA（Vue Router） |
|---|---|
| 多个 `.html` 文件 | 一个 html，多个组件 |
| `location.href` 整页刷新 | `router.push` 局部切换，不刷新 |
| 跳转白屏、重新加载 | 无缝切换、保留状态 |

## 二、安装与注册

前面 `npm create vue` 时已经选了 Router（依赖已装好）。现在在入口注册：

```js
// src/main.js
import { createApp } from 'vue'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import App from './App.vue'
import router from './router'      // ← 引入路由

const app = createApp(App)
app.use(ElementPlus)
app.use(router)                     // ← 注册路由
app.mount('#app')
```

## 三、定义路由表

新建 `src/router/index.js`，告诉 Router："什么网址，显示什么组件"：

```js
import { createRouter, createWebHistory } from 'vue-router'
import UserList from '../views/UserList.vue'
import UserDetail from '../views/UserDetail.vue'

// 路由表：地址 → 组件
const routes = [
  { path: '/',             name: 'list',   component: UserList },
  { path: '/users/:id',    name: 'detail', component: UserDetail }
]

const router = createRouter({
  history: createWebHistory(),    // 用 HTML5 history 模式（URL 干净，没 #）
  routes
})

export default router
```

- `path`：网址路径；
- `component`：访问这个路径时显示的组件；
- `:id` 是**动态参数**（下面讲）。

## 四、`router-view`：页面出口

`App.vue` 里放一个 `<router-view />`——它是"页面出口"，**当前路由对应的组件会显示在这里**：

```vue
<template>
  <div class="layout">
    <header class="topbar">用户管理系统</header>
    <main class="content">
      <router-view />     <!-- 路由匹配的组件（UserList / UserDetail）显示在这 -->
    </main>
  </div>
</template>
```

访问 `/` → `<router-view />` 里显示 `UserList`；访问 `/users/3` → 显示 `UserDetail`。顶栏和布局始终不变，只中间内容换——这就是 SPA。

---

## 五、`router-link` / `router.push`：怎么跳转

**声明式**（像 `<a>` 标签）：

```vue
<router-link to="/">返回列表</router-link>
```

**编程式**（在 JS 里跳，更常用）：

```js
import { useRouter } from 'vue-router'
const router = useRouter()

function goDetail(row) {
  router.push(`/users/${row.id}`)    // 跳到详情页
}
```

| jQuery | Vue Router |
|---|---|
| `location.href = '/detail.html'`（整页刷新） | `router.push('/users/3')`（局部切换） |
| `<a href="detail.html">` | `<router-link to="/">` |

## 六、动态路由参数 `:id`

详情页要展示"某一个"用户，但用户有 10 个。不可能给每个用户写一条路由。用**动态参数**：

```js
{ path: '/users/:id', component: UserDetail }
```

`:id` 是占位符。访问 `/users/3` 时，`id` 就是 `3`；访问 `/users/7` 时，`id` 就是 `7`。**一条路由，匹配所有用户**。

在 `UserDetail.vue` 里取这个参数：

```js
import { useRoute } from 'vue-router'
const route = useRoute()
const id = route.params.id    // 当前 URL 里的 :id（如 '3'）
```

---

## 七、`useRoute` 与 `useRouter`

两个容易混的 API，分清：

| API | 干什么 | 类比 |
|---|---|---|
| `useRoute()` | 读**当前路由信息**（参数、路径等） | "我现在在哪页" |
| `useRouter()` | 拿到路由器，**做跳转** | "带我去某页" |

```js
const route  = useRoute()    // 读：route.params.id
const router = useRouter()   // 跳：router.push(...) / router.back()
```

> 记忆：`route` 比 `router` 少个 r，是"路由（当前那一条）"；`router` 是"路由器（执行跳转的工具）"。

## 八、详情页实战

新建 `src/views/UserDetail.vue`：取 `:id` → 请求该用户 → 展示 → 提供返回：

```vue
<script setup>
import { ref, onMounted } from 'vue'
import { useRoute, useRouter } from 'vue-router'
import { getUserById } from '../api/user'

const route = useRoute()
const router = useRouter()

const user = ref(null)

onMounted(async () => {
  user.value = await getUserById(route.params.id)   // 用 URL 里的 id 请求数据
})
</script>

<template>
  <div class="page">
    <el-page-header @back="router.back()" content="用户详情" />
    <el-descriptions v-if="user" :column="1" border>
      <el-descriptions-item label="姓名">{{ user.name }}</el-descriptions-item>
      <el-descriptions-item label="邮箱">{{ user.email }}</el-descriptions-item>
      <!-- ... -->
    </el-descriptions>
  </div>
</template>
```

列表页加个"详情"入口（`UserList.vue`）：

```js
import { useRouter } from 'vue-router'
const router = useRouter()
function handleDetail(row) {
  router.push(`/users/${row.id}`)
}
```

```vue
<el-button size="small" link @click="handleDetail(row)">详情</el-button>
```

`api/user.js` 加单查接口：

```js
export function getUserById(id) {
  return request.get(`/users/${id}`)
}
```

现在点列表的"详情" → URL 变成 `/users/3` → 详情页加载该用户 → 点"返回"回到列表。**整个过程不刷新页面**，这就是 SPA。

## 🏋️ 小练习

1. **404 页面**：加一条路由 `{ path: '/:pathMatch(.*)*', component: NotFound }`，做一个简单的"页面不存在"组件。
2. **路由高亮**：在顶栏加一个 `<router-link to="/">用户管理</router-link>`，注意它默认有激活样式。
