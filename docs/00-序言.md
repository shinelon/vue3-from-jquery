# 序言：为什么 jQuery 程序员要学 Vue

你现在用 jQuery 干活得心应手——选元素、绑事件、发请求、操作 DOM，闭着眼睛都能写。那为什么要花力气学 Vue？

这一篇不讲代码，只讲一件事：**jQuery 和 Vue，到底差在哪？**想通这一点，后面所有章节你都会有"原来如此"的感觉。

---

## 一、命令式 vs 声明式：最大的思维转变

这是 jQuery 和 Vue 最本质的区别。

用 jQuery，你是在**指挥浏览器**：先拿到 DOM 元素，再一步步告诉它该怎么变。这种风格叫**命令式**——你"命令"页面每一步做什么。

举个例子：点按钮，让一个数字 +1，并显示出来。

**jQuery 写法：**

```js
let count = 0
$('#btn').on('click', function () {
  count++
  $('#count').text(count)   // ← 手动找到元素，手动更新文字
})
```

你要操心三件事：

1. 数据存在哪（`count` 变量）；
2. 怎么响应事件（`on('click')`）；
3. **数据变了之后，怎么把变化同步到页面上**（`$('#count').text(count)`）。

第 3 件是最烦的。业务一旦复杂，一个数据变了，你就要**记得**去更新所有依赖它的 DOM。忘更新一处，bug 就来了——"明明数据改了，页面怎么没变？"这是 jQuery 时代最常见的 bug 来源。

**Vue 写法：**

```vue
<script setup>
import { ref } from 'vue'
const count = ref(0)        // 数据
</script>

<template>
  <button @click="count++">加一</button>
  <span>{{ count }}</span>  <!-- 自动同步，不用手动改 -->
</template>
```

你只操心两件事：**数据**（`count`）和**事件**（`@click`）。数据一变，**页面自动更新**——你再也不用手动去改 DOM 了。Vue 替你盯着数据，数据一变，它自动把页面上对应的地方刷新。

这种风格叫**声明式**：你"声明"页面应该长什么样（`<template>` 部分），而不是"命令"它怎么变。你只管数据，视图交给 Vue。

> 📌 **一句话记住：jQuery 是"你操作 DOM"，Vue 是"你操作数据，Vue 替你操作 DOM"。**

## 二、组件化：告别复制粘贴

用 jQuery 写过一个"用户卡片"，要在 5 个页面用？你大概率是**复制粘贴 5 份** HTML + 5 份 JS。哪天要改个样式，得改 5 处，漏一处就不一致——这种坑你一定踩过。

Vue 的**组件**，把 HTML、JS（甚至 CSS）打包成一个**可复用的零件**：

```vue
<!-- UserCard.vue：一个自包含的组件 -->
<template>
  <div class="card">
    <img :src="user.avatar" />
    <h3>{{ user.name }}</h3>
  </div>
</template>

<script setup>
defineProps(['user'])   // 接收外部传入的 user 数据
</script>
```

用的时候，像写 HTML 标签一样：

```vue
<UserCard :user="alice" />
<UserCard :user="bob" />
```

**改一处，处处生效。** 这是现代前端开发的基石。组件化让你把复杂页面拆成一块块小积木，每块积木独立维护、自由组合。

## 三、生态：你不是一个人在战斗

jQuery 时代，做一个稍大的项目，你要自己找轮播插件、自己处理页面跳转、自己管登录状态、自己拼接各种库……东拼西凑出一套"自己的方案"。

Vue 有成熟、统一的官方生态，各司其职：

| 你要做的事 | jQuery 时代 | Vue 生态 |
|---|---|---|
| 多页面跳转 | 自己写 location.href / 多个 html | **Vue Router** |
| 全局状态（登录态等） | 全局变量 / cookie | **Pinia** |
| 项目构建/打包 | 手动管理 script 顺序 | **Vite**（秒级启动） |
| 表格/表单/弹窗 | 找各种 jQuery 插件 | **Element Plus** 等组件库 |
| 发请求 | `$.ajax` | **axios** |

你要做的 CRUD 后台，90% 的零件都有现成的、经过千万项目验证的方案，你只需要把它们**组装**起来。

---

## 四、所以，值得学吗？

- **工作上**：新项目几乎都用框架（Vue / React），纯 jQuery 的新项目越来越少。学会 Vue，你的路会宽很多。
- **思维上**：一旦理解了"声明式"和"组件化"，以后学 React、Angular 都是同一套思想，触类旁通——这是终身受用的底层思维。
- **并不难**：核心其实就一句话——**"操作数据，而不是操作 DOM"**。抓住这点，剩下的都是细节，熟能生巧。
