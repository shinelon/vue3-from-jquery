# 第 2 章 · 响应式：告别手动改 DOM

上一章你已经尝到"数据一变，页面自动更新"的甜头。这一章我们把它讲透：**Vue 是怎么做到的？`ref` 和 `reactive` 有什么区别？**

> 📂 可运行 demo 在 [`demos/ch02-reactivity/`](../demos/ch02-reactivity/)。

---

## 一、先回忆 jQuery 的痛苦

jQuery 时代，"数据变了 → 更新页面"是**手动活**：

```js
var user = { name: '张三', age: 20 }

// 显示到页面
$('#name').text(user.name)
$('#age').text(user.age)

// 用户改名了
user.name = '李四'
$('#name').text(user.name)   // ← 必须手动再设一次！忘了页面就不变
```

**痛在哪？** 数据和页面是割裂的。数据改了，你必须**记得**去同步对应的 DOM。业务复杂时，一处数据可能影响多处 DOM，漏一处就是 bug。

Vue 的**响应式**，就是要把这个"手动同步"彻底自动化。

---

## 二、响应式到底是什么

一句话：**某些特殊的变量，你一改它，Vue 自动把页面上所有用到它的地方全部更新。**

这种"特殊变量"在 Vue 里用 `ref` 或 `reactive` 创建。第 1 章我们用了 `ref`，这一章讲清楚它和 `reactive` 的区别。

---

## 三、`ref`：包裹任意值

```js
import { ref } from 'vue'

const count = ref(0)         // 数字
const name = ref('张三')     // 字符串
const visible = ref(false)   // 布尔
const list = ref([])         // 数组（连对象也行）
```

`ref` 能包**任何类型**的值。

⚠️ **读写规则（新手必记）：**

- **JS 里**：读写都要加 `.value` → `count.value`、`count.value = 1`
- **模板里**：直接用，**不加** `.value` → `{{ count }}`

```vue
<template>
  <p>{{ count }}</p>              <!-- 模板里不加 .value -->
  <button @click="count++">加</button>
</template>

<script>
const { ref } = Vue
const count = ref(0)
function inc() {
  count.value++                   <!-- JS 里要加 .value -->
}
</script>
```

> **为什么 JS 里要 `.value`？**
> 因为 `ref(0)` 实际返回的是一个"盒子"对象 `{ value: 0 }`。Vue 通过监视这个盒子的 `.value` 属性，才能在它变化时收到通知、触发更新。数字、字符串这类**基本类型**没法直接被监视，所以得用盒子包一层。对象 `reactive` 能直接监视，所以不用盒子（见下一节）。

---

## 四、`reactive`：包裹对象

`ref` 是"包单个值"，`reactive` 是"包整个对象"，而且**不用 `.value`**：

```js
import { reactive } from 'vue'

const user = reactive({
  name: '张三',
  age: 20
})

user.name = '李四'    // 直接改，不用 .value，页面自动更新
user.age++
```

对比一下，同样一个 user 对象，用 `ref` 写：

```js
const user = ref({ name: '张三', age: 20 })
user.value.name = '李四'   // ← 要 .value
```

---

## 五、`ref` 还是 `reactive`？怎么选

给你一个**新手最省心的经验法则**：

- **基本类型**（数字、字符串、布尔）→ 只能用 `ref`；
- **对象** → 两者都行，但：
  - 整个对象经常被**替换**（`user = 新对象`）→ 用 `ref`；
  - 只是**改对象里的属性** → `reactive` 更顺手（不用 `.value`）。

> 🟢 **本教程的建议：默认用 `ref`，简单统一。**
>
> `reactive` 有两个坑容易绊倒新手：
> 1. **解构会失去响应式**：`const { name } = user` 之后，`name` 就不再自动更新了；
> 2. **不能整体替换**：`user = { 新对象 }` 会丢掉响应式（要逐个属性赋值）。
>
> `ref` 没这些坑，所以新手"无脑 ref"最省心。等熟练了，再按场景挑。

---

## 六、`computed`：派生数据（自动计算）

很多值是"算出来"的：总价 = 单价 × 数量、剩余任务数 = 总数 − 完成数。jQuery 里你得**每次手动重算 + 手动更新 DOM**；Vue 用 `computed` 一步到位：

```js
import { ref, computed } from 'vue'

const price = ref(10)
const quantity = ref(3)

// total 会自动等于 price × quantity，依赖一变就自动重算
const total = computed(() => price.value * quantity.value)
```

`computed` 两个特点：
1. **自动跟踪依赖**：你在回调里用了哪些 ref，它就盯着哪些；
2. **有缓存**：依赖没变时，多次读取只算一次（比每次手动调函数高效）。

对照 jQuery：你得自己写个 `updateTotal()`，**每次** price 或 quantity 变都**记得调用**——又是"手动同步"的老问题。

---

## 七、动手看完整例子

打开 [`demos/ch02-reactivity/index.html`](../demos/ch02-reactivity/index.html)，核心代码：

```js
const { createApp, ref, computed } = Vue

createApp({
  setup() {
    const price = ref(10)        // 单价
    const quantity = ref(1)      // 数量
    const vip = ref(false)       // 是否会员

    // 总价 = 单价 × 数量 × 折扣，全自动
    const total = computed(() => {
      const discount = vip.value ? 0.8 : 1
      return price.value * quantity.value * discount
    })

    return { price, quantity, vip, total }
  }
}).mount('#app')
```

试试改数量、勾选会员——**你从没碰过 DOM，但总价自己就变了**。这就是响应式 + computed 的威力。

## 🏋️ 小练习

1. **加库存上限**：数量不能超过 99，也不能小于 1。
   - 提示：把 `quantity++` 换成一个函数，里面判断边界。
2. **再加一个折扣码**：输入框输入 code，如果等于 `VUE123` 就打五折，叠加在会员折扣之外。
   - 提示：再加一个 `ref` 和一段 computed 逻辑。
