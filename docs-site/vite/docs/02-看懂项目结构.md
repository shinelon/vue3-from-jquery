# 第 2 章 · 看懂 Vite 项目结构

上一章我们建好了项目、跑起来了。但脚手架生成了一堆文件，它们各管什么？这一章把项目结构讲透——尤其要理解一个跟裸 HTML 时代**完全不同**的关键点：**Vite 里，`index.html` 是入口**。

> 📂 本章仍对应分支 `vite/01-init`。

---

## 一、项目目录全景

```
vite-notes/
├── node_modules/      ← npm 装的依赖（别手动改，别提交 git）
├── public/            ← 静态资源（图标等，原样拷贝，第 7 章细讲）
├── src/
│   ├── assets/        ← 会被构建处理的资源（图片等，第 7 章）
│   ├── components/    ← 你的组件放这里
│   ├── App.vue        ← 根组件（主战场）
│   └── main.js        ← 入口文件
├── index.html         ← 页面入口（重点！见下）
├── package.json       ← 项目"身份证"：依赖清单 + 脚本
└── vite.config.js     ← Vite 配置（后续章节的主战场）
```

---

## 二、最大的思维转变：`index.html` 是入口

裸 HTML 时代，`index.html` 是"页面主体"，里面塞满内容。**Vite 时代，`index.html` 变成了"入口清单"**——它很干净，只声明"挂载点"和"第一个 JS 文件"：

```html
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <title>vite-notes · 笔记应用</title>
  </head>
  <body>
    <div id="app"></div>                         <!-- Vue 挂载点 -->
    <script type="module" src="/src/main.js">    <!-- 入口 JS，从这里开始 -->
    </script>
  </body>
</html>
```

> 🔑 **关键**：浏览器先加载 `index.html`，看到 `<script type="module" src="/src/main.js">`，于是去请求 `main.js`；`main.js` 里 `import App from './App.vue'`，浏览器再去请求 `App.vue`……**这就是上一章说的"按需请求"的起点**。一切都从 `index.html` 这个入口展开。

这跟 webpack 时代（入口写在 `webpack.config.js` 里）不同——Vite 把入口回归到 `index.html`，更直观，也更符合浏览器的原生机制。

---

## 三、`src/main.js`：入口脚本

```js
import { createApp } from 'vue'   // 从 vue 引入 createApp
import App from './App.vue'       // 引入根组件

createApp(App).mount('#app')      // 创建应用，挂载到 index.html 的 #app
```

三件事：引入 Vue、引入根组件、挂载。整个应用的代码，都是从这里 `import` 出去的一张大网。

---

## 四、`src/App.vue`：根组件，主战场

你写代码最多的地方。一个 `.vue` 文件分三段（template 结构 / script 逻辑 / style 样式），这是 Vue 单文件组件（SFC）的标准写法，主教程有详细讲，这里不重复。

---

## 五、`package.json`：项目的身份证

```json
{
  "name": "vite-notes",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": { "vue": "..." },        // 运行时依赖
  "devDependencies": { "vite": "...", "@vitejs/plugin-vue": "..." }  // 开发时依赖
}
```

两个重点：
- **`scripts`**：`npm run xxx` 实际跑的命令。`dev` 跑 `vite`，`build` 跑 `vite build`。
- **`dependencies` vs `devDependencies`**：前者是**运行时**要用的（如 vue，上线后还在用）；后者是**开发时**才用的（如 vite，上线后不需要）。`npm install xxx` 默认进 dependencies，`npm install -D xxx` 进 devDependencies。

---

## 六、`vite.config.js`：后续章节的主战场

现在它很简陋：

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()]    // 让 Vite 认识 .vue 文件
})
```

但从下一章起，这里会逐步长出各种配置：路径别名、环境变量、开发代理、构建选项、部署 base……**本教程后面大半内容，都是在丰富这个文件**。先记住它的位置。

---

## 对照：项目结构这件事

| | 裸 HTML 时代 | Vite 工程化 |
|---|---|---|
| 入口 | index.html 塞满内容 | index.html 只声明挂载点 + 入口 JS |
| 文件组织 | 全塞一个 html | `src/` 分文件 + `import` 串联 |
| 依赖记录 | 没有 | `package.json` 清单 |
| 配置 | 没有 | `vite.config.js` 集中配置 |

---

## 🏋️ 小练习

1. 在 `src/components/` 下新建一个 `Hello.vue`，在 `App.vue` 里 `import` 并用它（复习组件化）；
2. 打开 `package.json`，找到 `devDependencies` 里的 `vite`，记下版本号——它就是你装的 Vite 版本。
