# 第 11 章 · 库模式：打包一个组件库

前面我们一直用 Vite 构建**应用**（一个完整的网站）。但 Vite 还能干另一件事：**打包一个库**（library）——比如你写了个 Vue 组件，想发布到 npm 给别人用。这一章用**库模式**把一个组件打包成可发布的产物。

> 📂 配套 demo 在 [`demos/lib-mode/`](../demos/lib-mode/)，一个把 `MyButton` 组件打包成库的最小示例。

---

## 一、应用模式 vs 库模式

| | 应用模式（前面所有章节） | 库模式（本章） |
|---|---|---|
| 产物 | 一个完整的网站（index.html + JS/CSS） | 一个**库文件**（给别人 import 用） |
| 入口 | index.html | 库的入口 JS |
| 用 `build` | `vite build` | `vite build` + `build.lib` 配置 |
| 谁用 | 浏览器直接访问 | 别的项目 `npm install` 后 import |

---

## 二、配置 `build.lib`

库模式的核心是 `vite.config.js` 里配 `build.lib`：

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  build: {
    lib: {
      entry: 'src/index.js',          // 库的入口
      name: 'LibModeDemo',            // UMD 模式下的全局变量名
      formats: ['es', 'umd'],         // 输出 ES 和 UMD 两种格式
      fileName: 'lib-mode'            // 输出文件名
    },
    rollupOptions: {
      external: ['vue']               // vue 不打进库（用你库的人自己有 vue）
    }
  }
})
```

两个关键点：
- **`formats: ['es', 'umd']`**：ES 给现代打包工具用，UMD 给 `<script>` 直接引入用；
- **`external: ['vue']`**：把 vue 排除在库之外。因为用你库的项目肯定已经有 vue，你别把 vue 也打进去（会重复、变大）。

---

## 三、库的入口：`src/index.js`

```js
import MyButton from './MyButton.vue'

export { MyButton }          // 具名导出
export default { MyButton }  // 默认导出
```

这是库的"门面"——别人 `import { MyButton } from 'your-lib'` 时，拿到的就是这里导出的东西。

---

## 四、打包看看

```bash
cd demos/lib-mode
npm install
npm run build
```

产物在 `dist/`（Vite 8 实测）：
- `lib-mode.js`（ES 格式，给打包工具用）
- `lib-mode.umd.cjs`（UMD 格式，给浏览器 `<script>` 或 CommonJS）
- `lib-mode.css`（组件样式）

> 💡 **关于警告**：库模式第一次 build 可能提示 `MIXED_EXPORTS` 或 `MISSING_GLOBAL_NAME`。前者是因为同时用了具名和默认导出（配 `output.exports: 'named'` 解决）；后者是 UMD 模式没告诉它 vue 的全局名（配 `output.globals: { vue: 'Vue' }` 解决）。本 demo 的 `vite.config.js` 已配好这两项。

这就是一个**可发布的库**！别人 `npm install` 后能 `import { MyButton }` 用。

---

## 五、发布到 npm（了解流程）

打包好就能发布：
```bash
npm login          # 登录 npm 账号
npm publish        # 发布（dist/ 会上传）
```

> ⚠️ 发布前要配好 `package.json` 的 `name`（不能重名）、`version`、`main` / `module`（指向 dist 产物）、`peerDependencies`（声明需要使用者自备 vue）。这是库开发的标准流程，这里点到为止。

## 🏋️ 小练习

1. 在 lib-mode 里再加一个组件（如 `MyInput.vue`），在 `index.js` 导出，重新 build 看产物；
2. 在另一个项目里 `npm install` 这个库（用 `npm link` 或本地路径），`import { MyButton }` 用上它。
