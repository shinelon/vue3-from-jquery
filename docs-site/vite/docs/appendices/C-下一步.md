# 附录 C · 下一步学什么

恭喜读完整本 Vite 教程！你已经能用 Vite 独立搭项目、配工程、打包部署、浅尝库 / 插件 / SSR。这一页给你更细的学习路径，按需挑选。

---

## 🪜 优先级最高的几步

### 1. 把 vite-notes 玩熟
改它的 vite.config，逐项试（别名、代理、环境变量、build）。每个配置都动手改一遍，比读十遍有用。

### 2. 把 Vite 用到真实项目
用 Vite + Vue 搭一个你自己个小项目（博客、待办、工具站），完整走一遍开发 → 部署。

### 3. 学 TypeScript + Vite
给 Vite 项目上 TS（主教程第 12 章有迁移实战）。类型系统能帮你少踩很多 bug。

---

## 🚀 进阶方向（按兴趣挑）

### 工程化
- **Nuxt**：Vue + Vite 的全栈 / SSR 框架，开箱即用，要做正经产品首选；
- **Monorepo**（pnpm workspace）：多包项目管理；
- **CI/CD**：GitHub Actions 自动构建部署。

### 深入 Vite
- 读 `unplugin-auto-import`、`@vitejs/plugin-vue` 源码，理解插件 / 编译内部；
- 写一个有用的 Vite 插件发布 npm；
- 研究 Environment API（Vite 6+ 的新架构）。

### 前端广度
- **React + Vite**：换框架不换工具，`npm create vite` 选 react；
- **UI 库深度**：Element Plus / Naive UI 的主题定制、按需引入；
- **性能优化**：路由懒加载、图片优化、Web Vitals。

## 🧭 思维上的下一步

你已经完成了关键转变：**从"操作 DOM / 裸 HTML"到"用构建工具系统化开发"**。带着这套思维：

- 学 **React / Next.js**：构建理念相通，迁移很快；
- 关注 **Rust 化**：Vite(Rolldown) / Turbopack / Rspack，工具链在变快；
- 关注 **AI 辅助开发**：结合 Claude Code 等工具提效。
