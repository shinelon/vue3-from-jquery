# 第 8 章 · 插件机制初识：Vite 的扩展点

你大概注意到了，`vite.config.js` 里有一行 `plugins: [vue()]`。这个 `vue()` 就是插件。Vite 的强大，很大程度来自插件生态。这一章初识插件机制——搞懂它是什么、`plugin-vue` 干了啥、生态里有哪些常用插件。**本章挂在 `vite/05-assets` 分支（纯认知 + 示例，不落库新配置）。**

> 📂 本章挂在分支 `vite/05-assets`（不落库新配置，避免污染累积项目）。

---

## 一、插件是什么

一句话：**插件 = 给 Vite 加能力的扩展**。

Vite 核心"只管"编译模块、起 dev、打包。但很多事核心不管，交给插件：
- 认识 `.vue` 文件？—— `@vitejs/plugin-vue` 插件；
- 自动导入 API（不用每个文件写 `import { ref }`）？—— `unplugin-auto-import`；
- 自动注册组件（不用每个组件手写 import）？—— `unplugin-vue-components`；
- 压缩图片、生成 SVG 雪碧图、mock 数据……都有插件。

`vite.config.js` 的 `plugins` 数组，就是你"开启哪些扩展"的清单。

---

## 二、`@vitejs/plugin-vue` 到底干了啥

我们一直在用 `vue()`，它做了最关键的一件事：**让 Vite 认识 `.vue` 文件**。

没有它，Vite 看到 `App.vue` 不知道怎么处理（它默认只认 `.js` / `.ts` / `.css` 等）。`plugin-vue` 注册了一个 **transform** 钩子：当 Vite 遇到 `.vue` 文件时，把它编译成浏览器能跑的 JS（template → render 函数、script → JS、style → CSS 注入）。

> 🔑 可以理解为：**plugin-vue 是 ".vue 文件的翻译官"**。没它，Vite 不懂 Vue。

这也是为什么第 1 章建项目选 Vue 模板时，脚手架自动给你装了它、配进了 `plugins`。

---

## 三、插件怎么用（以自动导入为例）

讲个很受欢迎的插件：`unplugin-auto-import`。它让你**不用每个文件写 `import { ref } from 'vue'`**，自动帮你导入。

装：
```bash
npm install -D unplugin-auto-import
```

配：
```js
// vite.config.js
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({ imports: ['vue'] })   // ← 自动导入 vue 的 API
  ]
})
```

配好后，代码里直接写 `ref`、`computed`、`onMounted`，**不用 import**，插件自动补上。

> ⚠️ 本教程的 vite-notes 项目**不启用**这类插件——我们要让你看清每个 `import`，建立扎实基础。等熟练了，再追求"少写 import"的便利。这里只是让你见识插件能干嘛。

---

## 四、和它配套的 `unplugin-vue-components`

另一个常见搭档：自动注册组件。配好后，`<NoteItem />` 直接用，**不用写 `import NoteItem from ...`**，插件自动按组件名找到并注册（配合 Element Plus 等 UI 库特别爽，按需引入）。

这两个 unplugin 是现代 Vue 项目的"便利套装"，但你得先理解手动 import（本教程的方式），才知道自动导入省了什么。

---

## 五、去哪找插件

- **Vite 官方插件**：[vite.dev/plugins](https://vite.dev/plugins/)；
- **awesome-vite**：GitHub 上的插件合集；
- **unplugin 系列**：一套跨构建工具（Vite/webpack/Rollup）的通用插件；
- UI 库自带：Element Plus、Naive UI 等都有对应的 Vite 插件。

> 💡 装插件的标准流程：`npm i -D xxx` → 在 `vite.config.js` 的 `plugins` 数组加一行 → 重启 dev。

---

## 六、插件背后：Rollup 插件接口（剧透第 12 章）

Vite 插件本质是 **Rollup 插件的超集**——一个返回 `{ name, transform(), load(), ... }` 的对象。Vite 在此基础上加了 dev 专属钩子（如 HMR 相关）。

这意味着：你会写 Vite 插件，也大致懂了 Rollup 插件。**第 12 章**我们会亲手写一个最小的自定义插件，揭开它的内部结构。

---

## 对照：扩展能力这件事

| | 裸 HTML 时代 | Vite 插件机制 |
|---|---|---|
| 加能力 | 找各种 jQuery 插件，手动 `<script>` 引入 | `npm i -D` + `plugins` 数组一行 |
| 生态 | jQuery 插件（已式微） | Vite/Rollup/unplugin 大生态 |
| 标准化 | 各搞各的 | 统一的插件接口 |
