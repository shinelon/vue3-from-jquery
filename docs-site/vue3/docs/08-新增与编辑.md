## 一、先看 jQuery 怎么做表单（复习痛点）

jQuery 时代提交一个表单，你要手动干一堆活：

```js
$('#save').on('click', function () {
  var name = $('#name').val()
  var email = $('#email').val()
  // 1. 手动逐个取值
  // 2. 手动逐个校验
  if (!name) { alert('请输入姓名'); return }
  if (!/^[\w.]+@[\w.]+$/.test(email)) { alert('邮箱格式错'); return }
  // 3. 手动提交
  $.ajax({ url: '/users', type: 'POST', data: { name, email }, success: function(){ ... } })
})
```

痛点：**逐个取值、逐个 if 判断、手动 alert、手动拼 data**。字段一多就又长又乱，校验逻辑散落各处。

Vue + Element Plus 怎样？**`v-model` 自动收集数据 + 一份 `rules` 声明校验规则 + 一个函数提交**。

## 二、`el-form` + `rules`：声明式表单校验

核心三步：**双向绑定数据、声明校验规则、提交时统一校验**。

```vue
<script setup>
import { ref } from 'vue'

const formRef = ref()                              // 表单引用，用来调 validate()
const form = ref({ name: '', email: '', phone: '' })

// 校验规则：声明"每个字段要满足什么"
const rules = {
  name:  [{ required: true, message: '请输入姓名', trigger: 'blur' }],
  email: [
    { required: true, message: '请输入邮箱', trigger: 'blur' },
    { type: 'email', message: '邮箱格式不正确', trigger: 'blur' }
  ],
  phone: [{ pattern: /^1\d{10}$/, message: '手机号格式不正确', trigger: 'blur' }]
}

async function submit() {
  try {
    await formRef.value.validate()   // 统一校验，不通过会 reject
  } catch {
    return                            // 校验失败，停下
  }
  // 校验通过，提交 form.value ...
}
</script>

<template>
  <el-form ref="formRef" :model="form" :rules="rules" label-width="80px">
    <el-form-item label="姓名" prop="name">
      <el-input v-model="form.name" />
    </el-form-item>
    <el-form-item label="邮箱" prop="email">
      <el-input v-model="form.email" />
    </el-form-item>
  </el-form>
</template>
```

几个要点：

- `:model="form"`：表单数据对象；`:rules="rules"`：校验规则；
- `<el-form-item prop="name">`：`prop` 指向 `form` 的哪个字段，校验才知道管谁；
- `v-model="form.name"`：输入自动写回 `form.name`——**不用手动取值了**；
- `formRef.value.validate()`：提交时一次性校验全部字段，不通过的自动在对应输入框下红字提示——**不用手动 alert 了**。

| jQuery | Vue + el-form |
|---|---|
| `$('#x').val()` 逐个取值 | `v-model` 自动收集 |
| 逐个 `if` + `alert` | 一份 `rules` 声明式校验 |
| 错误提示自己写 | 校验失败自动红字提示 |

`trigger: 'blur'` 表示失焦时校验，也可以 `'change'`（输入即校验）。

## 三、`el-dialog`：弹窗

新增/编辑通常用弹窗承载表单。Element Plus 的 `<el-dialog>`：

```vue
<el-dialog v-model="visible" title="新增用户" width="480px">
  <!-- 表单放这里 -->
  <template #footer>
    <el-button @click="visible = false">取消</el-button>
    <el-button type="primary" @click="submit">保存</el-button>
  </template>
</el-dialog>
```

`visible` 为 `true` 弹窗显示，`false` 关闭。`#footer` 插槽放底部按钮（第 5 章讲的 slot）。

## 四、组件 `v-model`：父子控制弹窗显隐

我们把弹窗表单**抽成一个独立组件 `UserDialog.vue`**（第 5 章的组件化）。父组件（列表页）要能控制它的开关。怎么控制？

用**组件上的 `v-model`**。在父组件写：

```vue
<UserDialog v-model="dialogVisible" />
```

它等价于：

```vue
<UserDialog
  :model-value="dialogVisible"
  @update:model-value="dialogVisible = $event"
/>
```

而子组件 `UserDialog` 里：

```js
const props = defineProps({ modelValue: Boolean })       // 接收开关状态
const emit = defineEmits(['update:modelValue'])          // 声明会修改它

function close() {
  emit('update:modelValue', false)    // 关闭：通知父把 dialogVisible 改成 false
}
```

这就是"组件上的 `v-model`"——**父组件一个变量，双向控制子组件**。规则固定：子组件用 `modelValue` 接、`update:modelValue` 抛。

## 五、新增 / 编辑共用一个表单

新增和编辑的表单长得**一模一样**，区别只是：新增是空表单，编辑要填入现有数据。我们用**一个 `UserDialog` 搞定两者**，靠一个 `editing` prop 区分：

- `editing` 为 `null` → 新增模式（空表单）；
- `editing` 为某个用户对象 → 编辑模式（填入该用户数据）。

父组件控制：

```js
function handleAdd() {
  editing.value = null          // 新增
  dialogVisible.value = true
}
function handleEdit(row) {
  editing.value = row           // 编辑这一行
  dialogVisible.value = true
}
```

## 六、`watch`：弹窗打开时初始化数据

弹窗打开时（`modelValue` 变成 `true`），根据是新增还是编辑，初始化表单数据。用 `watch` 监听：

```js
import { watch } from 'vue'

watch(() => props.modelValue, (visible) => {
  if (!visible) return
  if (props.editing) {
    form.value = { ...props.editing }     // 编辑：复制现有数据
  } else {
    form.value = { name: '', email: '', phone: '', role: 'viewer', status: 1 }  // 新增：默认值
  }
})
```

`watch(要监听的, 回调)`：被监听的值一变，回调就执行。`{ ...props.editing }` 是用展开运算符复制对象（ES6 速查讲过），避免直接改原对象。

提交时，新增走 `POST`，编辑走 `PUT`：

```js
if (props.editing) {
  await updateUser(form.value.id, form.value)   // PUT /users/:id
} else {
  await addUser(form.value)                      // POST /users
}
emit('success')     // 通知父组件：保存成功了，去刷新列表
close()
```

## 七、接回列表页

`UserList.vue` 把上一章的占位按钮接上：

```vue
<script setup>
import UserDialog from '../components/UserDialog.vue'

const dialogVisible = ref(false)
const editing = ref(null)

function handleAdd()  { editing.value = null;  dialogVisible.value = true }
function handleEdit(row) { editing.value = row; dialogVisible.value = true }
</script>

<template>
  <!-- 工具栏的按钮 -->
  <el-button type="primary" @click="handleAdd">+ 新增用户</el-button>
  <!-- 表格操作列 -->
  <el-button size="small" @click="handleEdit(row)">编辑</el-button>

  <!-- 弹窗：v-model 控制开关，@success 监听保存成功 -->
  <UserDialog v-model="dialogVisible" :editing="editing" @success="loadData" />
</template>
```

`api/user.js` 补两个接口：

```js
export function addUser(data) {
  return request.post('/users', data)
}
export function updateUser(id, data) {
  return request.put(`/users/${id}`, data)
}
```
