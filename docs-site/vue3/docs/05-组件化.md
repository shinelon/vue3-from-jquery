# 第 5 章 · 组件化：拆分与复用

序言里提过：jQuery 时代复用 UI 靠"复制粘贴 5 份"，改一处要改 5 处。Vue 用**组件**解决这个问题。这一章讲怎么把页面拆成可复用的零件，以及零件之间怎么通信。

这是过渡到 CRUD 项目前的最后一块基础，**也是 Vue 真正发挥威力的地方**。

> 📂 可运行 demo 在 [`demos/ch05-components/`](../demos/ch05-components/)，一个用户列表，每张卡片是一个独立组件。

---

## 一、jQuery 的复用之痛（再戳一次）

要做一个"用户卡片"，5 个页面都要用：

- **复制粘贴**：拷 5 份 HTML + JS，改样式改 5 处，漏一处就不一致；
- **jQuery 插件**：能封装，但要写 `$.fn.userCard = function(opts){...}`，配置复杂、调试难；
- 数据变了，还得手动调 `render()` 重画。

Vue 的组件：**一个 `.vue` 文件就是一个可复用的"积木"**，数据变了自动更新，改一处处处生效。

---

## 二、定义一个组件

新建 `src/components/UserCard.vue`，它就是一个组件：

```vue
<script setup>
const props = defineProps({
  user: { type: Object, required: true }   // 声明接收一个 user 属性
})
</script>

<template>
  <div class="card">
    <div class="avatar">{{ user.name.charAt(0) }}</div>
    <div class="info">
      <h3>{{ user.name }}</h3>
      <p>{{ user.email }}</p>
    </div>
  </div>
</template>

<style scoped>
.card { display: flex; align-items: center; gap: 12px; border: 1px solid #eee; padding: 12px; border-radius: 6px; }
.avatar { width: 48px; height: 48px; border-radius: 50%; background: #42b883; color: #fff; display: flex; align-items: center; justify-content: center; font-weight: bold; }
</style>
```

---

## 三、在父组件里用它

在 `App.vue` 里：

```vue
<script setup>
import UserCard from './components/UserCard.vue'   // ← import 进来

const alice = { name: '张三', email: 'zhangsan@x.com' }
const bob   = { name: '李四', email: 'lisi@x.com' }
</script>

<template>
  <UserCard :user="alice" />     <!-- ← 当 HTML 标签用，:user 把数据传进去 -->
  <UserCard :user="bob" />
</template>
```

**三步**：`import` → 当标签 `<UserCard />` 用 → `:user="..."` 传数据。

| jQuery | Vue |
|---|---|
| 写个 `renderUserCard(user)` 函数拼 HTML，调用 5 次 | 写个 `.vue` 组件，`<UserCard :user="x" />` 用 5 次 |

差别在于：Vue 组件是**活的**——数据变了，卡片**自动更新**，不用手动重画。

---

## 四、`props`：父传子（像函数参数）

`props` 是父组件传给子组件的数据。**子组件只能读，不能直接改**（单向数据流）。

```vue
<!-- 子组件 -->
<script setup>
defineProps({
  user: { type: Object, required: true },
  count: { type: Number, default: 0 },        // 带默认值
  title: { type: String, required: true }     // 必填
})
</script>
```

这就像函数声明参数：`function UserCard(user, count = 0, title)`。

- 父传：`<UserCard :user="alice" :count="3" title="管理员" />`
- 子用：模板里 `{{ user.name }}`、`{{ count }}`

> ⚠️ **props 是只读的**。子组件里 `user.name = 'x'` 是禁止的——要改，得用下面的 `emit` 通知父组件改。

---

## 五、`emit`：子传父（像回调函数）

子组件要"通知"父组件（比如"用户点了删除"），用 `emit` 触发事件：

```vue
<!-- 子组件 UserCard.vue -->
<script setup>
const props = defineProps({ user: Object })
const emit = defineEmits(['delete'])      // 声明会触发哪些事件

function handleDelete() {
  emit('delete', props.user.id)           // 触发 delete 事件，带上 id
}
</script>

<template>
  <button @click="handleDelete">删除</button>
</template>
```

父组件**监听**这个事件：

```vue
<!-- 父组件 App.vue -->
<template>
  <UserCard :user="alice" @delete="handleDelete" />   <!-- ← @delete 监听 -->
</template>

<script setup>
function handleDelete(id) {
  // 父组件决定怎么删（数据归父管）
  users.value = users.value.filter(u => u.id !== id)
}
</script>
```

| jQuery | Vue |
|---|---|
| 子模块调一个回调 `onDelete(id)` | 子组件 `emit('delete', id)`，父组件 `@delete` 监听 |

形式上是"事件"，本质是回调，但更解耦。

---

## 六、为什么搞得这么"绕"？（理解单向数据流）

jQuery 老哥一定会问：**子组件为啥不能直接改数据、直接删用户，非要绕一圈 emit 给父？**

答：**单向数据流**——数据归**父组件**管，子组件只负责"报告发生了什么"（emit），由父决定"怎么处理"。

打个比方：员工（子组件）不能自己去改公司数据库，得**打报告**（emit）给主管（父组件），主管批准了才改。听起来繁琐，但这样：

- **数据流向永远是"父 → 子"**，清晰可追踪；
- 一个数据只在"一个地方"被修改，不会到处乱改；
- 项目一大，这种纪律能救命。

反过来，jQuery 那种"谁都能改 DOM、谁都能改数据"的自由，恰恰是大型项目 bug 不断的根源。

## 七、`slot`：插槽（给组件塞一段内容）

`props` 传**数据**，`slot` 传**一段模板/HTML**。

```vue
<!-- 容器组件 Card.vue -->
<template>
  <div class="card">
    <slot />        <!-- ← 占位：父组件塞的内容会出现在这 -->
  </div>
</template>
```

```vue
<!-- 使用 -->
<Card>
  <h3>任意标题</h3>
  <p>任意内容，甚至另一个组件……</p>
</Card>
```

`<slot />` 就是个**占位符**，父组件决定占位处放什么。CRUD 项目里弹窗、卡片这些"容器型"组件会大量用到 slot。

## 八、动手看完整例子

打开 [`demos/ch05-components/`](../demos/ch05-components/)，关键代码：

**`UserCard.vue`（子组件）**——接收 `user`（props），点击删除时 `emit('delete', id)`：

```vue
<script setup>
const props = defineProps({ user: { type: Object, required: true } })
const emit = defineEmits(['delete'])
function remove() {
  emit('delete', props.user.id)
}
</script>

<template>
  <div class="card">
    <div class="avatar">{{ user.name.charAt(0) }}</div>
    <div class="info">
      <h3>{{ user.name }}</h3>
      <p>{{ user.email }}</p>
    </div>
    <button class="del" @click="remove">删除</button>
  </div>
</template>
```

**`App.vue`（父组件）**——`v-for` 渲染卡片，监听 `@delete`：

```vue
<script setup>
import { ref } from 'vue'
import UserCard from './components/UserCard.vue'

const users = ref([
  { id: 1, name: '张三', email: 'zhangsan@example.com' },
  { id: 2, name: '李四', email: 'lisi@example.com' },
  { id: 3, name: '王五', email: 'wangwu@example.com' }
])

function handleDelete(id) {
  users.value = users.value.filter(u => u.id !== id)   // 数据归父管
}
</script>

<template>
  <UserCard
    v-for="u in users"
    :key="u.id"
    :user="u"
    @delete="handleDelete"
  />
</template>
```

点删除按钮 → 子组件 `emit('delete', id)` → 父组件 `handleDelete` 改数据 → 列表自动更新。**整个链路里，删除逻辑只在父组件一处**，这就是单向数据流的好处。

2. **用 slot**：给 `UserCard` 底部加一个 `<slot name="footer" />`，父组件传入自定义内容（如"管理员"标签）。
