# 前置：必须懂的 ES6 极简速查

Vue3 的代码里会用到一些"现代 JS"语法。如果你平时还在写 `var`、`function`、字符串拼接 `+`，这份速查能帮你快速对上号。

**不用全背**，看懂、能照着抄就行。后面章节真正用到时，我还会再提一句。把它当字典，忘了就回来翻。

---

## 1. `let` / `const`（替代 `var`）

```js
// 老写法
var count = 0
var name = 'Vue'

// 新写法
let count = 0        // 会变的，用 let
const name = 'Vue'   // 不会重新赋值的，用 const
```

- `let`：变量，能重新赋值（`count = 1`）。
- `const`：常量，不能重新赋值；但如果它装的是对象，**对象内部的属性还能改**。

> Vue 代码里几乎不用 `var`。默认用 `const`，确实需要改值时才用 `let`。

---

## 2. 箭头函数（替代 `function`）

```js
// 老写法
$('#btn').on('click', function (e) { /* ... */ })
arr.map(function (item) { return item.id })

// 新写法
$('#btn').on('click', (e) => { /* ... */ })
arr.map(item => item.id)   // 只有一句时，可省略 {} 和 return
```

箭头函数更短，还顺带解决了 `this` 指向的老大难问题。Vue 里的事件处理、回调几乎全是箭头函数。

---

## 3. 解构（从对象 / 数组里取值）

```js
const user = { name: '张三', age: 20, role: 'admin' }

// 老写法
const name = user.name
const age = user.age

// 新写法：对象解构
const { name, age } = user

// 数组解构
const [first, second] = [10, 20]
```

> Vue 里 `defineProps` 解构、函数参数解构到处都是，看到 `const { a, b } = xxx` 就是这意思。

---

## 4. 模板字符串（替代字符串拼接）

```js
const name = 'Vue'
const age = 3

// 老写法：一堆 + 号
const msg = 'Hello ' + name + ', 你 ' + age + ' 岁了'

// 新写法：反引号 ` ` 包裹，${ } 里写变量
const msg = `Hello ${name}, 你 ${age} 岁了`
```

注意是**反引号** `` ` ``（键盘左上角，~ 那个键），不是单引号。

---

## 5. 展开运算符 `...`

```js
// 合并数组
const a = [1, 2]
const b = [3, 4]
const c = [...a, ...b]   // [1, 2, 3, 4]

// 合并 / 复制对象
const defaults = { page: 1, size: 10 }
const params = { ...defaults, keyword: 'vue' }
// 结果：{ page: 1, size: 10, keyword: 'vue' }
```

> `...` 就是把数组/对象"摊开"塞进去。Vue 里合并配置、传 props 时常用。

---

## 6. `Promise` + `async / await`（异步，对接后端用）

jQuery 用 `$.ajax` + 回调；现代 JS 用 `async / await`，**像写同步代码一样**处理异步。

```js
// 老写法（回调地狱）
$.get('/api/users', function (data) {
  console.log(data)
})

// 新写法（async/await）
async function loadUsers() {
  const res = await axios.get('/api/users')  // ← 等！等结果回来再往下走
  console.log(res.data)
}
```

- 函数前加 `async`，里面才能用 `await`；
- `await` 会**暂停**，等右边那个异步操作（请求）出结果，再继续往下。

> 比回调清晰太多了。第 7 章对接接口会详细讲。

## 7. ES Module：`import` / `export`（Vite 项目必备）

jQuery 靠 `<script>` 标签按顺序加载，文件多了顺序错乱就出 bug。现代 JS 用 `import / export` 显式声明依赖。

```js
// utils.js —— 导出（对外提供）
export function add(a, b) { return a + b }   // 具名导出
export default function main() { /* ... */ } // 默认导出（一个文件最多一个）

// app.js —— 导入（拿来用）
import main, { add } from './utils.js'
//     ↑默认     ↑具名（要加花括号）
```

> 第 4 章引入 Vite 后会天天写这个。现在只要知道"这是 JS 的模块化"就行。

---

## 8. 可选链 `?.` 和 空值合并 `??`

现代代码里很常见，看不懂容易懵，特意拎出来讲：

```js
// 可选链 ?. ：安全地取深层属性，中间是 null/undefined 也不会报错
const city = user?.address?.city
// 等价于老写法：
// const city = user && user.address && user.address.city

// 空值合并 ?? ：只有 null / undefined 时才用默认值
const page = query.page ?? 1
// （注意：?? 只认 null/undefined；|| 会把 0、'' 也当 falsy，容易踩坑）
```

> 这两个能帮你少写一堆 `if` 判空。看到 `?.` 就读成"如果有就取、没有就算了"。
