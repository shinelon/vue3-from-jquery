# 第 6 章 · HMR 热更新原理：凭什么秒级

从第 1 章起你就在用 HMR——改代码不刷新整页、瞬间见效。但你可能一直没想明白：它到底怎么做到的？这一章把 HMR 的原理讲透。**本章不写新代码，仍用 `vite/04-proxy` 分支演示。**

> 📂 本章挂在分支 `vite/04-proxy`（不新增文件，纯讲原理）。

---

## 一、先回顾：没有 HMR 的世界

裸 HTML 时代，改完代码要 **F5 整页刷新**：
- 浏览器重新请求 `index.html` → 重新下载所有 JS/CSS → 重新执行 → 页面状态（你填的表单、点的计数）**全丢**。

痛点：慢（重新加载一切）、断（状态丢失，调试到一半归零）。

---

## 二、HMR 的核心思想：只换改动的部分

**HMR（Hot Module Replacement，热模块替换）** 的目标是：**改了哪个模块，就只替换那一个，不刷新整页，其他模块的状态保留。**

比如你改了 `NoteItem.vue`，HMR 只把 `NoteItem` 这个组件**重新编译、精确替换**进页面，`App.vue` 的 `notes` 数据、你勾选的状态——**原封不动**。

> 🔑 对照：F5 是"重启整台机器"，HMR 是"热插拔一个零件"。

---

## 三、Vite 的 HMR 怎么工作（三步）

### 第 1 步：监听文件变化

Vite dev 服务器用 **chokidar** 监听项目文件。你一保存 `NoteItem.vue`，服务器立刻收到"文件变了"的通知。

### 第 2 步：重新编译那一个模块

服务器只重新编译**变了的那个模块**（`NoteItem.vue` → 新的 JS），不影响其他文件。

### 第 3 步：通过 WebSocket 通知浏览器替换

- dev 服务器和浏览器之间有一条 **WebSocket** 长连接（你在第 1 章 curl 时看到的 `/@vite/client` 就是它）；
- 服务器通过 WebSocket 告诉浏览器："`NoteItem.vue` 变了，新的代码是 xxx"；
- 浏览器收到后，**卸载旧模块、装入新模块**，精确更新 UI。

整个过程**毫秒级**，因为：只动一个文件、用 WebSocket 推送（不用浏览器轮询）、按需编译（不打包）。

---

## 四、为什么 Vite 的 HMR 特别快

回忆第 0 章的三个快的原因，HMR 用到了两个：

1. **No-Bundle**：HMR 只编译改动的那一个模块，不重新打包整个项目；
2. **精确失效**：Vite 知道"哪个模块 import 了改动的模块"，只更新受影响的边界，不像 webpack 常常整页 invalidate。

> 💡 你可以做个实验：在 `App.vue` 里把 `notes` 初始值改成 5 条、保存——页面瞬间变 5 条。再改 `NoteItem.vue` 的样式、保存——样式瞬间变。这就是 HMR 在精确替换。

---

## 五、HMR 的边界：什么时候会"整页刷新"

HMR 不是万能的。以下情况 Vite 会**回退到整页刷新**（live reload）：

- 改了**没有 HMR 边界**的文件（比如改了普通的 `.js` 工具函数，且没有模块声明如何热替换它）；
- 改了 `vite.config.js` 或 `package.json`（要重启 dev）；
- 改了 `.env`（环境变量，要重启）；
- HMR 替换失败（比如新旧模块结构差异太大）。

> 📌 所以你会偶尔看到"改了某个文件，页面整页刷新了"——那是 HMR 判断无法精确替换，回退了。`.vue` 组件几乎总能精确 HMR，所以改组件体验最好。

---

## 六、自己接受 HMR 事件（进阶，了解即可）

框架（Vue / React）已经帮你处理好了 HMR 边界，你通常不用管。但 Vite 也暴露了 API，让你在纯 JS 模块里手动接受 HMR：

```js
// 某个模块.js
if (import.meta.hot) {
  import.meta.hot.accept((newModule) => {
    // 这个模块被热替换时的回调
  })
}
```

日常用 `.vue` 不需要写这个（plugin-vue 帮你做了）。知道有这东西即可。

| | 裸 HTML（F5） | Vite HMR |
|---|---|---|
| 更新范围 | 整页 | 改动的单个模块 |
| 状态 | 全丢 | 保留 |
| 速度 | 慢（重新加载一切） | 毫秒级 |
| 机制 | 浏览器重新请求 | WebSocket 推送 + 按需编译 |
