# 第 3 章 · 路径别名：告别 ../../../

项目一长大，你就会遇到一个烦人事：组件 A 在 `src/views/user/Profile.vue`，它要用 `src/components/UserCard.vue`，import 得写成：

```js
import UserCard from '../../components/UserCard.vue'   // 数几个点？
```

层级一深，`../../../../../` 数到眼花，文件一移动全得改。这一章用 Vite 的**路径别名**，一行配置彻底告别。

> 📂 配套项目 `project/vite-notes/`，本章对应分支 `vite/02-alias`。

---

## 一、问题：相对路径地狱

裸 HTML 时代你没这烦恼（全塞一个文件）。但工程化后文件分层，相对路径就成了负担：

```js
// 在 src/views/user/Profile.vue 里
import UserCard from '../../components/UserCard.vue'   // 退两层
import { fetchUser } from '../../../api/user.js'        // 退三层
import { formatDate } from '../../../../utils/date.js'  // 退四层……
```

痛点：
1. **数点数到眼花**，多一层少一层都报错；
2. **文件一移动**，所有 import 路径全得改；
3. **不直观**，看不出引用的是 `src` 下哪个东西。

---

## 二、解决：给 `src` 起个别名 `@`

思路很简单：给 `src` 目录起个**别名 `@`**，以后无论在多深的目录，`@/xxx` 都直接指向 `src/xxx`。

配置在 `vite.config.js`：

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { fileURLToPath, URL } from 'node:url'   // ← 新增

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      // '@' 指向项目的 src 目录
      '@': fileURLToPath(new URL('./src', import.meta.url))
    }
  }
})
```

> 💡 `fileURLToPath(new URL('./src', import.meta.url))` 这串看着长，作用就是把 `'./src'` 转成绝对路径。这是社区主流写法（`create-vue` 脚手架也这么写），比直接写 `'@': '/src'` 更可靠。

配好后，前面那一坨 `../../` 全变成：

```js
import UserCard from '@/components/UserCard.vue'   // 清爽
import { fetchUser } from '@/api/user.js'
import { formatDate } from '@/utils/date.js'
```

**无论你在多深的目录，`@` 永远指向 `src`**，不用数点，文件移动也不影响（只要还在 src 下）。

---

## 三、在 vite-notes 里实操

看 `vite/02-alias` 分支。我们把第 1 章内联的列表项，抽成一个 `NoteItem` 组件，并在 `App.vue` 里用 `@` 引入。

新增 `src/components/NoteItem.vue`（一个笔记项组件）：

```vue
<script setup>
defineProps({
  note: { type: Object, required: true }
})
</script>

<template>
  <li>
    <input type="checkbox" v-model="note.done" />
    <span :class="{ done: note.done }">{{ note.title }}</span>
  </li>
</template>
```

`src/App.vue` 改成用 `@` 引入它：

```vue
<script setup>
import { ref } from 'vue'
import NoteItem from '@/components/NoteItem.vue'   // ← 别名，不用 ./

const notes = ref([ /* ... */ ])
</script>

<template>
  <h1>📝 vite-notes</h1>
  <ul>
    <NoteItem v-for="n in notes" :key="n.id" :note="n" />
  </ul>
</template>
```

> 🔑 这里 `App.vue` 在 `src/` 根，用 `./components/NoteItem.vue` 也行。但**养成用 `@` 的习惯**，等组件挪到 `src/views/xxx/` 深层时，就体会到它的好了。

---

## 四、编辑器也要认识 `@`（否则没提示）

配好 Vite 后，**运行时** `@` 能用了。但你的编辑器（VS Code）可能还不认识 `@`，导致**跳转不了、没有自动补全**。需要再告诉编辑器一声——在项目根加个 `jsconfig.json`：

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": { "@/*": ["./src/*"] }
  }
}
```

> 💡 这个文件只给**编辑器**看（让 VS Code 认识 `@`），不影响 Vite 构建。两处配置（vite.config 给运行时、jsconfig 给编辑器）都配上，体验才完整。

---

## 对照：路径引用这件事

| | 裸 HTML 时代 | 相对路径 | Vite 别名 `@` |
|---|---|---|---|
| 引入方式 | 全塞一个文件 | `../../components/X.vue` | `@/components/X.vue` |
| 文件移动 | — | 所有 import 要改 | 不影响（只要在 src 下） |
| 可读性 | — | 要数点 | 一眼看出在 src 下 |
