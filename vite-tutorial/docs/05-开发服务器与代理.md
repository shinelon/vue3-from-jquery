# 第 5 章 · 开发服务器与代理：解决跨域

开发时，你的前端跑在 `localhost:5173`，后端接口在 `localhost:3000`。你从前端发请求到后端，浏览器大概率报错：

```
Access-Control-Allow-Origin ... 跨域（CORS）被拦
```

这是浏览器的**同源策略**在拦你。这一章讲 Vite 怎么用**开发代理**优雅绕过它。

> 📂 配套项目 `project/vite-notes/`，本章对应分支 `vite/04-proxy`。

---

## 一、为什么会被拦：同源策略

浏览器有个安全规则叫**同源策略**：用 `fetch` / `axios` 请求时，如果目标地址和当前页面的**协议+域名+端口**任一不同，浏览器就拦截（除非后端明确允许，即 CORS 响应头）。

- 页面：`http://localhost:5173`
- 接口：`http://localhost:3000`

端口不同 → 不同源 → 被拦。

裸 HTML 时代你也遇到过：用 `$.ajax` 请求别的域，报 CORS 错。当时的解法往往是"求后端加 CORS 头"或"用 JSONP"。

---

## 二、解法：让 Vite 当中间人（开发代理）

思路：**别让浏览器直接请求后端，让 Vite 开发服务器当中间人转发。**

- 前端请求 `/api/notes`（同源，浏览器不拦）；
- Vite 收到 `/api/notes`，**代理转发**给 `http://localhost:3000/notes`；
- 后端响应，Vite 转回前端。

浏览器全程只跟 `localhost:5173`（Vite）打交道，**不存在跨域**。这是开发期最省事的方案。

> 🔑 对比上线：上线后前后端通常部署在同域（或用 nginx 反向代理），生产环境没这问题。**开发代理只是开发期的便利**，跟上线无关。

---

## 三、配置 `server.proxy`

在 `vite.config.js` 加 `server.proxy`：

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { fileURLToPath, URL } from 'node:url'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: { '@': fileURLToPath(new URL('./src', import.meta.url)) }
  },
  server: {
    port: 5173,
    proxy: {
      '/api': {                                       // 匹配以 /api 开头的请求
        target: 'http://localhost:3000',              // 转发目标（json-server）
        changeOrigin: true,                           // 改 Host 头，让后端以为是直接访问
        rewrite: (path) => path.replace(/^\/api/, '') // 去掉 /api 前缀：/api/notes → /notes
      }
    }
  }
})
```

三个关键字段：
- **`target`**：转发到哪（后端地址）；
- **`changeOrigin`**：通常 `true`，修改请求头里的 Host；
- **`rewrite`**：改写路径。这里把 `/api/notes` 变成 `/notes`（因为 json-server 的接口是 `/notes`，没有 `/api`）。

---

## 四、axios 配合：`baseURL: '/api'`

前端用 axios 时，把 `baseURL` 设成 `/api`，所有请求自动加 `/api` 前缀，正好被代理捕获：

```js
// src/api/request.js
import axios from 'axios'

const request = axios.create({
  baseURL: '/api',     // ← 所有请求以 /api 开头，走代理
  timeout: 10000
})

export default request
```

```js
// src/api/note.js
import request from './request'

export function getNotes() {
  return request.get('/notes')   // 实际请求 /api/notes → 代理转发 → :3000/notes
}
```

## 五、在 vite-notes 里实操

看 `vite/04-proxy` 分支。要做两件事：起一个假后端、让前端请求它。

**1. 起 json-server 假后端**（端口 3000）：

```bash
npm run server    # 跑 package.json 里的 "json-server --watch db.json --port 3000"
```

它会基于 `db.json` 提供 REST 接口：`GET /notes` 返回所有笔记。

**2. 前端请求它**：开**另一个终端**跑 `npm run dev`（5173）。`App.vue` 挂载时拉数据：

```vue
<script setup>
import { ref, onMounted } from 'vue'
import { getNotes } from '@/api/note'

const notes = ref([])
onMounted(async () => {
  notes.value = await getNotes()   // 走代理拉数据
})
</script>
```

> 💡 这里要开**两个终端**：一个 `npm run server`（后端），一个 `npm run dev`（前端）。这是开发常态。

打开 `localhost:5173`，笔记列表从 `db.json` 来了。打开 F12 → Network，看 `/api/notes` 请求——状态 200，说明代理通了。

---

## 六、多个后端 / 更复杂的代理

实际项目可能有多个后端。`proxy` 可以配多个键：

```js
proxy: {
  '/api': { target: 'http://localhost:3000', changeOrigin: true, rewrite: ... },
  '/upload': { target: 'http://localhost:4000', changeOrigin: true }   // 另一个后端
}
```

`proxy` 底层用的是 `http-proxy`，还支持 WebSocket（`ws: true`）、正则匹配、事件钩子等，进阶需求查文档即可。

---

## 对照：跨域这件事

| | 裸 HTML 时代 | Vite 开发代理 |
|---|---|---|
| 跨域解法 | 求后端加 CORS / JSONP | 前端配 `server.proxy`，免打扰后端 |
| 适用阶段 | 全程 | 仅开发期（上线靠同域部署 / nginx） |
| 配置位置 | 后端代码 | 前端 `vite.config.js` |
