# 小结 · 把 vite-notes 部署上线

恭喜！上篇 10 章走完，你从"裸 HTML 撞墙"一路走到"用 Vite 把一个应用搭到能上线"。这一篇把整个流程串起来，给你一张全景图。

## 全流程回顾：vite-notes 长大的 8 步

| 步骤 | 章节 | 你学会的 Vite 能力 |
|---|---|---|
| 1 | 第 1~2 章 | `npm create vite`、HMR、看懂项目结构（index.html 是入口） |
| 2 | 第 3 章 | `resolve.alias` 路径别名，告别 `../../../` |
| 3 | 第 4 章 | `.env` + `import.meta.env`，dev/线上不同配置 |
| 4 | 第 5 章 | `server.proxy` 开发代理，解决跨域 |
| 5 | 第 6 章 | HMR 原理（No-Bundle + WebSocket + 按需编译） |
| 6 | 第 7~8 章 | 静态资源（public vs assets）、CSS、插件机制 |
| 7 | 第 9 章 | `vite build`、chunk 拆分、产物优化 |
| 8 | 第 10 章 | SPA 部署、刷新 404 坑、nginx fallback |

对应代码分支 `vite/01-init` → `vite/08-final`，`git checkout` 任意一步对照。

## 一张图：vite.config.js 的成长

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { fileURLToPath, URL } from 'node:url'

export default defineConfig({
  plugins: [vue()],                    // 第 1 章
  resolve: {                           // 第 3 章
    alias: { '@': fileURLToPath(new URL('./src', import.meta.url)) }
  },
  server: {                            // 第 5 章
    port: 5173,
    proxy: {
      '/api': { target: 'http://localhost:3000', changeOrigin: true, rewrite: p => p.replace(/^\/api/, '') }
    }
  },
  build: {                             // 第 9 章
    target: 'es2015',
    rollupOptions: { output: { manualChunks(id) { /* 按需拆 vue/axios */ } } }
  }
})
```

就这么一个文件，承载了 Vite 工程化的核心。
