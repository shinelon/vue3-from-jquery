# 第 12 章 · 自定义插件开发入门

第 8 章我们"用"过插件（plugin-vue、unplugin-auto-import）。这一章反过来——**自己写一个 Vite 插件**，揭开它的内部结构。你会发现它没那么神秘。

> 📂 配套 demo 在 [`demos/plugin-hello/`](../demos/plugin-hello/)，一个把代码里的 `__HELLO__` 替换成一句话的最小插件。

---

## 一、插件是什么（再精确一次）

一个 Vite 插件，本质就是一个**返回对象的函数**：

```js
export default function myPlugin(options) {
  return {
    name: 'my-plugin',     // 插件名（调试/报错时能看到）
    // 各种"钩子"：在构建的不同阶段被 Vite 调用
    transform(code, id) { /* ... */ },
    load(id) { /* ... */ },
    // ...
  }
}
```

那个对象里的 `transform`、`load` 等方法叫**钩子（hook）**——Vite 在构建流程的特定时机调用它们，插件借机"插一手"改东西。

> 📌 这个结构其实是 **Rollup 插件规范**。Vite 插件 = Rollup 插件 + 一些 Vite 专属钩子。所以你会写 Vite 插件，也大致懂 Rollup 插件。

---

## 二、最常用的钩子：`transform`

`transform(code, id)` 在**每个模块被处理时**调用，参数是模块的源码 `code` 和文件路径 `id`。你返回改写后的代码，就完成了"翻译"。

plugin-vue 就是靠它：遇到 `.vue`，把 SFC 源码 transform 成 JS。

我们来写个自己的：把代码里的 `__HELLO__` 占位符替换成一句话。

---

## 三、动手：`vite-plugin-hello`

看 [`demos/plugin-hello/plugins/vite-plugin-hello.js`](../demos/plugin-hello/plugins/vite-plugin-hello.js)：

```js
export default function helloPlugin(options = {}) {
  const greeting = options.greeting || '来自插件'
  return {
    name: 'vite-plugin-hello',
    transform(code, id) {
      // 只处理 .js 文件
      if (id.endsWith('.js')) {
        return code.replace(/__HELLO__/g, JSON.stringify(greeting))
      }
      // 不返回（undefined）= 不处理这个文件
    }
  }
}
```

逻辑很简单：对每个 `.js` 文件，把 `__HELLO__` 替换成 `"来自插件"`。

---

## 四、用上它

在 `vite.config.js` 引入并加进 plugins：

```js
import { defineConfig } from 'vite'
import hello from './plugins/vite-plugin-hello.js'

export default defineConfig({
  plugins: [hello()]
})
```

代码里用占位符（`src/main.js`）：

```js
const greeting = __HELLO__   // ← 原始代码（直接跑会报错：__HELLO__ 未定义）
document.querySelector('#app').textContent = greeting
```

跑 `npm run dev`，页面显示"来自插件"——插件在 transform 阶段把 `__HELLO__` 替换成了字符串，浏览器实际运行的是替换后的代码。

> 🔑 这就是插件的核心能力：**在源码到产物之间，插入一道"加工"**。压缩、自动导入、Mock、代码生成……本质都是 transform（或其它钩子）干的事。

---

## 五、传配置给插件

插件是个函数，接收 `options`：

```js
plugins: [hello({ greeting: '你好，我来自自定义插件' })]
```

这样 `__HELLO__` 会被替换成你传的话。这是插件可配置的标准模式。

---

## 六、其它常见钩子（了解）

除了 `transform`，还有：

| 钩子 | 时机 | 典型用途 |
|---|---|---|
| `config` | 读 vite.config 后 | 改 Vite 配置 |
| `resolveId` / `load` | 解析/加载模块时 | 自定义模块来源（如虚拟模块） |
| `transform` | 处理每个模块源码时 | 改代码（最常用） |
| `buildStart` / `buildEnd` | 构建开始/结束 | 初始化/清理 |
| `configureServer` | 配 dev 服务器 | 加自定义中间件（如 mock） |

> 💡 写复杂插件时查 [Vite 插件 API 文档](https://vite.dev/guide/api-plugin)。日常 80% 的需求用 `transform` 就够了。

---

## 对照：扩展工具能力

| | 裸 HTML 时代 | Vite 插件 |
|---|---|---|
| 给工具加功能 | 没有统一机制 | 写个返回对象的函数，配进 plugins |
| 介入时机 | — | 钩子在构建各阶段被调用 |
| 复用 | 自己写的处理逻辑难复用 | 插件可发布 npm，人人能用 |
