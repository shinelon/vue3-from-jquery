# 第 1 章 · 10 分钟建一个 Vite 项目

上一章我们想通了"为什么是 Vite"。这一章动手——10 分钟建一个真 Vite 项目，跑起来，亲手体验秒级启动和热更新。

> 📂 配套项目在 [`project/vite-notes/`](../project/vite-notes/)，本章对应分支 `vite/01-init`（脚手架基座）。

---

## 一、用官方脚手架建项目

Vite 提供了一个一键脚手架。打开终端（Windows 用 PowerShell 或 Git Bash），`cd` 到你想放项目的目录，执行：

```bash
npm create vite@latest
```

它会问你两件事：

```
Project name: … vite-notes        ← 项目名，我们叫 vite-notes
Select a framework: … Vue         ← 选 Vue（本教程示例用 Vue）
Select a variant: … JavaScript    ← 选纯 JS（先不上 TypeScript）
```

> 💡 如果你想一步到位、跳过问答：`npm create vite@latest vite-notes -- --template vue`

然后进项目、装依赖、启动：

```bash
cd vite-notes
npm install        # 安装依赖（第一次 1~2 分钟）
npm run dev        # 启动开发服务器
```

终端会几乎**瞬间**输出：

```
  VITE v8.x  ready in 300 ms

  ➜  Local:   http://localhost:5173/
```

浏览器打开 `http://localhost:5173/`，看到 Vue 欢迎页就成了。**回想上一章**：webpack 项目大点要等几十秒，Vite 几百毫秒——这就是 No-Bundle 的威力。

---

## 二、亲手见识热更新（HMR）

别关掉 dev 服务器。打开编辑器，找到 `src/App.vue`，把欢迎语改一个字、保存。**注意浏览器**：页面**没有整页刷新**，但你改的地方**瞬间变了**——这就是 HMR（热模块替换）。

再做个更直观的：在 `<script setup>` 里加个计数器：

```vue
<script setup>
import { ref } from 'vue'
const count = ref(0)        // 加这几行
</script>

<template>
  <button @click="count++">点了 {{ count }} 次</button>   <!-- 加这行 -->
  <!-- ...原有内容... -->
</template>
```

保存，浏览器瞬间出现按钮。**点几下让 count = 5**，再去改别的代码、保存——你会发现 **count 还是 5，没被重置**。这就是 HMR 的精髓：**只替换改动的部分，组件状态保留**。

> 🔑 对裸 HTML 老哥的对照：以前你改完代码要 F5 整页刷新，填了一半的表单全没；HMR 让你改代码像"换零件"，机器还在转。

## 三、停掉、重启、打包

| 命令 | 作用 |
|---|---|
| `npm run dev` | 启动开发服务器（热更新，开发时用） |
| `npm run build` | 打包生产版本（输出到 `dist/`，准备上线） |
| `npm run preview` | 本地预览打包后的结果（模拟上线效果） |

试试 `Ctrl+C` 停掉 dev，再跑 `npm run build`——你会看到生成了一个 `dist/` 目录，里面是压缩、优化过的文件，这就是要部署上线的成品。`npm run preview` 可以本地预览它。

## 四、我们这个教程的配套项目：vite-notes

为了让后面章节有真实场景，本教程围绕一个**笔记应用 vite-notes** 讲。它的 `vite/01-init` 分支，就是刚建好的最小基座（一个静态笔记列表）。

```bash
git checkout vite/01-init    # 看本章对应的代码状态
```

打开 [`project/vite-notes/src/App.vue`](../project/vite-notes/src/App.vue)，你会看到一个极简的笔记列表——先写死的几条数据。后续章节会逐步给它加配置（别名、环境变量、代理……），让它长成一个能上线的小应用。

> ⚠️ 磁盘上的 `project/vite-notes/` 默认是最终成品（`vite/08-final`）。要看某章的中间态，用 `git checkout vite/NN` 切到对应分支。

## 对照：建项目这件事

| | 裸 HTML 时代 | Vite 工程化 |
|---|---|---|
| 新建项目 | 手建 index.html + 手下 CDN | `npm create vite@latest` 一键 |
| 引入 Vue | `<script src="cdn/vue.js">` | 脚手架自动 `npm i vue` + 配好 |
| 启动 | 双击 html 用浏览器开 | `npm run dev`，秒级 ready |
| 改代码看效果 | F5 整页刷新 | HMR 局部热更 |

省掉的是"手搭脚手架、手管依赖、手动刷新"，换来的是"一条命令开工、改完即见"。
