# 第 14 章 · 生态与下一步：environments / Rolldown

正文到此结束。这一章作为收尾，讲讲 Vite 正在往哪走（environments、Rolldown），以及你接下来该学什么。

---

## 一、Vite 的演进：从 Rollup 到 Rolldown

你用的 Vite 8，相比早期版本有个大变化：**打包器从 Rollup 换成了 Rolldown**。

- **Rollup**：JS 写的老牌打包器，成熟但大项目偏慢；
- **Rolldown**：用 Rust 重写的打包器，**快得多**，能处理更大的项目。

Vite 8 起，Rolldown 默认集成（你在第 9 章 build 时看到的 `rolldown-runtime` 就是它）。对你写配置而言，`build.rollupOptions` 这套 API 依然兼容（少数边角差异，比如 `manualChunks` 要用函数形式——见第 9 章）。

> 📌 趋势：前端构建工具正在"Rust 化"（更快）。Vite 用 Rolldown，webpack 的继任者 Turbopack / Rspack 也是 Rust。你学的 Vite 配置理念不会过时，底层只会更快。

---

## 二、Environment API（了解）

Vite 6 引入、逐步稳定的 **Environment API**，是 Vite 内部架构的一次大升级。

以前 Vite 假设"一个项目 = 一个客户端环境"。但现代需求变复杂了：一个项目可能同时有**客户端 + 服务端 + Worker + 多种框架**（比如 SSR + RSC + Web Worker）。Environment API 让 Vite 能统一管理"多个构建环境"。

> 💡 对入门用户：**你几乎感知不到它**。它是给框架作者（Nuxt、Remix、Vite 插件开发者）用的底层能力。你日常的 vite.config 不用管它。知道有这东西、Vite 正变得更通用即可。

## 三、常用 Vite 插件清单（收藏）

实际项目里，这些插件很常用：

| 插件 | 作用 |
|---|---|
| `@vitejs/plugin-vue` | 让 Vite 认识 `.vue`（必装） |
| `unplugin-auto-import` | 自动导入 API，少写 import |
| `unplugin-vue-components` | 自动注册组件 |
| `unplugin-vue-router` | 基于文件的路由（仿 Nuxt） |
| `vite-plugin-svg-icons` | 把 SVG 当组件用 |
| `vite-plugin-pwa` | 给应用加 PWA / 离线能力 |
| `rollup-plugin-visualizer` | 打包体积可视化分析 |
| `vite-plugin-mock` | dev 时 mock 接口 |

> 🔍 去哪找：[awesome-vite](https://github.com/vitejs/awesome-vite)、[vite.dev/plugins](https://vite.dev/plugins/)。

## 五、你接下来该学什么

按优先级：

1. **把 vite-notes 玩熟**：改它的配置、加功能，把每项配置都试一遍；
2. **学 Vue 进阶**（如果你还没）：组合式函数、自定义指令、Teleport、Suspense——见主教程附录 C；
3. **学 TypeScript**：给 Vite 项目上 TS（主教程第 12 章有 Vue + TS 迁移实战）；
4. **学 Nuxt**：当你需要 SSR / 全栈，别手搭，用 Nuxt（Vite + Vue 的全栈框架）；
5. **学一个 UI 库的深度用法**：Element Plus / Naive UI / Ant Design Vue；
6. **读 Vite 插件源码**：挑一两个常用插件（如 unplugin-auto-import）读源码，加深对插件机制的理解。

## 六、思维上的下一步

你已经完成了关键的转变：**从"一个 html 写天下"到"用构建工具系统化开发"**。带着这套思维：

- 学 **React / Next.js**：Vite 能配 React（`npm create vite` 选 react），构建理念相通；
- 学 **其他构建工具**：webpack（老项目还在用）、Turbopack、Rspack——理解了 Vite，看这些会快；
- 关注 **Rust 化趋势**：前端工具链正在变快，原理值得跟进。
