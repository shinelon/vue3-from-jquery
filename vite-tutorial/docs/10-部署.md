# 第 10 章 · 部署：把 dist 推上线

`npm run build` 产出 `dist/`，这一章讲怎么把它推到线上——以及那个让无数新手栽跟头的 SPA"刷新 404"大坑。

> 📂 配套项目 `project/vite-notes/`，本章对应分支 `vite/07-deploy`（加了 vue-router，演示 SPA 部署）。

---

## 一、部署的本质：把 dist 丢到一个静态文件服务器

`dist/` 里全是静态文件（html/js/css/图片）。部署就是**把这些文件放到一个能被互联网访问的静态文件服务器**上。常见选择：

| 平台 | 特点 |
|---|---|
| **Vercel / Netlify** | 拖 dist 上去就行，自动 HTTPS、自动 SPA fallback，零配置 |
| **GitHub Pages** | 免费，适合个人项目 / demo |
| **nginx / 自己的服务器** | 完全可控，要自己配 |
| **对象存储**（OSS / COS / S3） | 便宜，配 CDN |

最简单的体验：用 Vercel / Netlify，把项目连上，指定 build 命令 `npm run build`、输出目录 `dist`，一键部署。

---

## 二、SPA 的"刷新 404"大坑（重点）

这是部署 SPA 最经典的坑。看 `vite/07-deploy`：我们加了 vue-router，用 **history 模式**，有 `/` 和 `/about` 两个路由。

部署后：
- 访问 `https://yoursite.com/` → 正常（服务器返回 index.html）；
- 从首页点链接进 `/about` → 正常（前端路由切换，不请求服务器）；
- **但在 `/about` 页面按 F5 刷新** → **404！**

为什么？因为刷新时浏览器**向服务器请求 `/about` 这个路径**，服务器去找 `/about` 这个文件——找不到（你的 dist 里只有 index.html，没有 about.html）→ 404。

> 🔑 根本原因：history 模式的 URL（`/about`）看起来像"服务器上的一个页面"，但其实是前端路由。服务器不知道这事，老老实实找文件，找不到就 404。

---

## 三、解决：让服务器把所有路径都指向 index.html

解法：**配置服务器，让任何找不到的路径都返回 index.html**（让前端路由接管）。各平台做法：

### Vercel / Netlify
**自动**处理 SPA fallback，零配置。这也是它们对新手友好的原因。

### nginx
加一行 `try_files`：
```nginx
location / {
  try_files $uri $uri/ /index.html;   # 找不到文件就回退到 index.html
}
```

### 对象存储 + CDN
通常配一个"404 重定向到 index.html"的规则。

> 📌 一句话：**只要服务器在找不到对应文件时回退到 index.html，SPA 的刷新 404 就解决了。**

---

## 四、`base`：部署在子路径时

如果你的站不在根域，而在子路径（如 `https://example.com/notes/`），资源会 404。这时在 `vite.config.js` 配 `base`：

```js
export default defineConfig({
  base: '/notes/',    // 所有资源 URL 前缀加 /notes/
  // ...
})
```

同时 vue-router 的 history 也要对应：`createWebHistory('/notes/')`。

> 💡 部署在根域（绝大多数情况）不用管 base，默认 `/` 即可。

---

## 五、本地预览 dist：`npm run preview`

上线前先本地预览打包结果：

```bash
npm run build      # 生成 dist
npm run preview    # 起一个服务器预览 dist
```

`preview` 起的服务器**默认带 SPA fallback**，所以本地预览不会 404（别被它骗了——真正部署到不支持 fallback 的服务器才会 404）。

## 对照：部署这件事

| | 裸 HTML 时代 | Vite SPA 部署 |
|---|---|---|
| 上线文件 | 源 html（多页面，各是独立 html） | `dist/` 单页应用（一个 index.html + JS 接管路由） |
| 刷新子页 | 正常（每个页面是真 html） | **会 404**（要配服务器 fallback） |
| 部署平台 | 任意静态服务器 | 同样，但要支持 SPA fallback |
